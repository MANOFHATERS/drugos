# v41 ROOT FIX (SEV2): the previous line had no ``python_version`` marker.
# Apache Airflow 2.x does NOT support Python 3.13+ (Airflow 2.10 added
# 3.12 support; 3.13 support is tracked in apache/airflow#39125 and
# slated for Airflow 3.0). Installing this requirements.txt on Python
# 3.13 would pull in Airflow 2.x and then fail at import time with
# cryptic ``AttributeError`` / ``ImportError`` errors. The fix adds the
# ``python_version < "3.13"`` marker so pip skips Airflow on Python 3.13+
# and prints a clear "no matching distribution" message instead. Copied
# from phase1/requirements.txt:49 (audit-2025 issue 38). When Airflow 3.0
# ships with 3.13 support, bump the version constraint and remove the
# marker.
apache-airflow>=2.8.0; python_version < "3.13"
